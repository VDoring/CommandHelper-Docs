조건문(Condition)

논리 구문

논리 구문은 조건문, 반복문 등과 같이 수학적인 원리를 적용하는 프로그램을 만들때 꼭 필요한 기본중의 기본이라고 할 수 있습니다.

이 파트에서는 논리 구문 중 주요 조건문을 알아보고, 이를 커맨드헬퍼에서 어떻게 사용하는지 알아보는 것이 주 목표입니다.

조건문(Condition)

조건문은 만약 어떠한 코드가 특정 조건을 만족하면 특정 코드를 실행하도록 하는 것입니다.

예를 들어, 변수 @a 와 @b 의 값이 같으면 msg('hello') 를 실행하고, 같지 않으면 msg('world') 을 실행할 수 있도록 하는 것이 조건문입니다.

if()

if() 구문은 조건문의 대표적인 펑션입니다. 인수로는 조건,코드[, 불일치 코드] 가 있습니다.

예제를 통해 if() 구문의 사용 방법을 알아보겠습니다.

예제 1.

/iftest $v = >>>
if($v == 'hello', msg('world'))
<<<

결과: /iftest hello 를 입력하면 world 라는 메시지가 출력되고, 그 외의 인수(/iftest 1, /iftest asdf) 를 입력하면 아무일도 일어나지 않음.

if 문의 첫번째 인수(조건 부분) 이 참일 경우에 두번째 인수(코드 부분) 의 코드를 실행합니다.

/iftest $v = >>>
if($v == 'hello', msg('world'), msg('bye'))
<<<

결과: /iftest hello 를 입력하면 world 라는 메시지가 출력되고, 그 외의 인수를 입력하면 bye 라는 메시지가 출력됨.

if 문의 첫번째 인수 부분이 거짓이면 두번째 인수(코드 부분) 을 실행하지 않고 세번째 인수(불일치 코드 부분) 이 실행됩니다.

예제 2.

/iftest $v = >>>
    if($v == 'hello') {
        msg('world')
    }
<<<

결과: /iftest hello 를 입력하면 world 라는 메시지가 출력되고, 그 외의 인수(/iftest 1, /iftest asdf) 를 입력하면 아무일도 일어나지 않음.

모든 펑션의 끝 부분 인수는 따로 떼어서 {} 로 묶어서 사용할 수도 있습니다. (예: tmsg(player()) { '메시지' })

/iftest $v = >>>
    if($v == 'hello') {
        msg('world')
    } else {
        msg('bye')
    }
<<<

결과: /iftest hello 를 입력하면 world 라는 메시지가 출력되고, 그 외의 인수를 입력하면 bye 라는 메시지가 출력됨.

불일치 코드를 쓰고 싶다면 if 문의 두번째 인수(코드 부분) 이 끝난 이후에 else 를 붙이고 또 {} 안에 코드를 쓰면 됩니다. (이러한 구문 형태는 if문만 지원합니다.)

예제 3.

/iftest $v $v2 = >>>
    if($v == 'hello') {
        if($v2 == 'world') {
            msg('hello world')
        } else {
            msg('hello bye')
        }
    }
<<<

결과: /iftest hello world 입력시 hello world 출력, /iftest hello asdf 입력시 hello bye 출력, /iftest asdf asdf 입력시 아무것도 출력하지 않음

if 구문 안에 if 문을 넣어도 당연히 동작합니다.(이론상으론 무한개)

/iftest $v = >>>
    if($v == 'hello') {
        msg('hello')
    } else if($v =='world') {
        msg('world')
    } else {
        msg('bye')
    }
<<<

결과: /iftest hello 입력시 hello 출력, /iftest world 입력시 world 출력, /iftest asdf 입력시 bye 출력

else 옆에 if() 를 또 붙여서 첫번째 조건이 일치하지 않지만 두번째 조건이 일치할 때와 같이 만들 수 있습니다.

/iftest $v = >>>
    if($v == 'hello') {
        msg('hello')
    } else {
        msg('world')
    } else {
        msg('bye')
    }
<<<

컴파일 에러: else 가 두 개 있습니다.

else 는 단 한개만 존재할 수 있습니다.

/iftest $v = >>>
    if($v == 'hello') {
        msg('hello')
    } else {
        msg('world')
    } else if($v =='world') {
        msg('bye')
    }
<<<

컴파일 에러: else 가 else if 앞에 있습니다.

else if 와 else 가 동시에 존재할 경우 else 는 항상 맨 뒤에 있어야 합니다.

switch()

switch() 펑션은 같은 변수를 값에 따라 각각 다른 작업을 실행해야 할 때 유용합니다.

switch() 구문은 다른 펑션들과는 달리 약간 특이한 문법을 사용합니다.

이 구문에 대해 예제를 통해 좀 더 알아보도록 하겠습니다.

예제 1.

/switchtest $ = >>>
    switch($) {
        case 'a':
            msg('Hello')
            break;
        case 'b':
            msg('World!')
            break;
        default:
            msg('Bye!')
            break;
    }
<<<

결과: 'a' 입력시 메시지 "Hello" 출력, 'b' 입력시 메시지 "World!" 출력, 그 이외 입력시 "Bye!" 출력

Switch 구문은 다른 구문과 다르게 문법이 상당히 다릅니다. 맨 위의 switch(변수) 를 통해 조건을 설정하고, 해당 변수의 결과를 {} 안에 case '값': 과 같은 형태로 지정하고,

case '값': 아래에 들여쓰기를 하여 코드를 적고, 마지막에 break; 구문을 통해서 구문을 끝마칩니다.

아직도 이해가 안되신다면 위 예제 switch() 구문을 if() 구문의 형태로 바꿔보도록 하겠습니다.

if() 구문의 형태
/switchtesttoif $ = >>>
    if($ == 'a') {
        msg('Hello')
    } else if($ == 'b') {
        msg('World!')
    } else {
        msg('Bye!')
    }
<<<

결과: 위 switch() 구문의 결과와 동일

쉽게 말해서 switch(변수) 구문에서의 case "값": 은 if() 구문에서의 if(변수 == 값) 과 같다고 볼 수 있고,

여러 case 들의 존재는 else if() 와 비슷하다고 볼 수 있으며, default: 는 else 와 같다고 볼 수 있습니다.

예제 2.

/switchtest $ = >>>
    switch($) {
        case 'a':
            msg('Hello')
        case 'b':
            msg('World!')
        default:
            msg('Bye!')
    }
<<<

결과: 'a' 입력시 메시지 "Hello", "World!", "Bye!" 출력, 'b' 입력시 메시지 "World!", "Bye!" 출력, 그 이외 입력시 "Bye!" 출력

switch() 구문이 if() 구문과 또 다른 점은 바로 break; 가 각 case 끝에 존재한다는 점입니다.

그렇다면 break; 가 존재하지 않는다면 어떨까요?

바로 다음 case 의 조건을 무시하고 실행시켜 버립니다.

즉 어떤 case 의 끝에 break 가 끝나지 않았다면 그 다음 case 의 구문도 실행하고, break; 가 나올 때까지 모든 조건을 무시하고 실행한다는 것이죠.

이것이 유용하게 쓰일 때도 있지만, 대부분 if() 구문을 간단하게 사용하기 위해 사용하기 때문에 break; 는 습관적으로 붙여 주시는 것이 필요합니다.
