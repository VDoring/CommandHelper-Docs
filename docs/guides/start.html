메뉴

시작하기

CommandHelper 는 Minecraft 를 위한 최고의 스크립팅 언어로, 간단한 구조와 변경을 재시작 없이 곧바로 서버에 적용할 수 있으며, 고급 프로그래밍 언어와 비슷한 기능을 지원하는 장점이 있는 스크립팅 플러그인입니다.

설치

이곳 에서 버전에 알맞은 커맨드헬퍼 플러그인을 다운로드 받으시고(최신 버전을 다운로드 받을 경우엔 full 버전으로 다운로드 받으세요.), 플러그인 폴더에 넣고 서버를 구동하세요.

위와 같은 아스키 아트가 서버에 뜨고, 밑에

[CommandHelper]: MethodScript files processed

[CommandHelper]: X alias(es) defined

라고 떴다면 커맨드헬퍼 설치가 완료된 것입니다. (설치가 되어도 버전에 따라 특정 기능이 구동되지 않는 경우도 있으므로 오류가 발생한다면 버전을 업데이트 하시거나 {커맨드 헬퍼는 항상 최신 버전을 완벽히 지원하므로 최신 버전을 사용하시는 것이 좋습니다.} 커맨드 헬퍼의 버전을 다운그레이드 하시는게 좋습니다.)

환경 설정하기

커맨드헬퍼는 CommandHelper 안의 .msa 나 .ms 확장자를 가진 파일을 수정하여 스크립트를 만들 수 있습니다.(Skript 의 .sk 와 유사)

이 .msa 나 .ms 파일의 수정은 일반적인 편집기를 통해 가능합니다.(메모장, Notepad++, Atom 등) 하지만 Notepad++ 편집기를 이용하면 구문 강조(Syntax Highlighter), 자동 완성 등등의 기능을 이용할 수 있습니다.

자세한 내용은 환경 설정 문서에서 설정한 후 이 문서로 돌아오세요.

폴더 살펴보기

커맨드헬퍼가 처음 실행되면 (플러그인 폴더)/CommandHelper 에 실행에 필요한 폴더와 파일들을 생성합니다. 이 문단에서는 생성되는 폴더와 파일에 대해 간단히 알아봅니다.

파일

aliases.msa

단축 명령어를 생성해주는 파일입니다.

이 파일은 일반 메모장과 동일하게 열 수 있습니다.

main.ms

서버가 시작될 때(혹은 커맨드헬퍼를 리로드할 때) 자동으로 실행되는 스크립트를 입력하는 파일입니다.

서버가 시작될 때부터 자동으로 실행되는 특징으로 인해 이벤트 처리에 많이 쓰입니다.

펑션 에 대한 기본적인 이해가 있어야 하므로 초보자 분들은 사용할 일이 거의 없습니다.

auto_include.ms

자동으로 커맨드헬퍼 상의 어느 파일에서도 include 할 수 있는 프로시저 를 만들 수 있는 파일입니다.

펑션 과 프로시저 에 대한 기본적인 이해가 있어야 합니다.

즉 초보자 분들은 main.ms 보다 더더욱 사용할 일이 없습니다.

persistence.db

영구 저장소 네트워크를 사용할 때 데이터가 저장되는 곳입니다.

커맨드헬퍼 명령줄 실행을 통해 내부를 볼 수 있습니다.

사용자가 굳이 건들 필요는 없는 파일입니다.

폴더

.cache

캐시 폴더입니다. 건들 필요는 없고 용량이 너무 커진다 싶으면 삭제해 주시는 것도 좋습니다.

다만 삭제할 시 성능에 문제가 생길 수 있으므로 유의해 주세요.

extensions

확장기능 을 넣는 폴더입니다.

includes

자동으로 include 되지 않는 프로시저 를 쉽게 분류하여 저장하라고 만들어진 폴더입니다.

프로시저 에 대한 이해가 없으면 사용할 일은 없습니다.

LocalPackages

커맨드헬퍼 언어로 이루어진 로컬 패키지(플러그인의 개념과 비슷함) 을 넣는 장소입니다. 해당 폴더 안에 임의의 폴더를 만들거나, 이미 만들어진 mslp 파일을 넣어서 서버에 적용할 수 있습니다.

자세한 정보는 로컬 패키지를 확인하세요.

logs

로그가 저장되는 폴더입니다.

prefs

설정 파일이 저장되는 폴더입니다.

자세한 정보는 고급 설정을 확인하세요.

첫번째 단축 명령어 만들기

단축 명령어란 어떠한 명령어를 다른 임의의 명령어를 입력했을 때 실행시키는 것입니다.

즉 만약 내가 /a 라는 명령어를 입력하면 /b 라는 명령어를 실행시키고 싶을 때, 단축 명령어 기능을 이용하는 것이죠.

단축 명령어 기능은 커헬에서 가장 기본적으로 지원하는 기능입니다.

단축 명령어를 만드려면, 앞에서 언급된 aliases.msa 파일을 통해 만들 수 있습니다.

aliases.msa 파일을 열고, 안에 적힌 모든 내용(예제이므로 필요가 없습니다) 을 지우고 파일에

/test = /say Test

라고 입력한 후 저장하세요.

저장한 후 서버 내(콘솔 혹은 OP 권한을 가진 마인크래프트 플레이어) 에서 /reloadalias 명령어를 입력하면

[CommandHelper] MethodScript file processed.
[CommandHelper] 1 alias(es) defined.

라는 메시지가 표시됩니다.

그리고 /test 라는 명령어를 입력하면 일반 마인크래프트의 /say Test 명령어를 실행한 것처럼 동작할 것입니다.
